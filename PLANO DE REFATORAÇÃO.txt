FASE 4 — PERFORMANCE MOBILE (repaint e scroll)
==========================================================

OBJETIVO: Corrigir o bug de conteúdo invisível ao trocar abas
e melhorar a fluidez geral no mobile.

TAREFAS:
1. No setInnerTab() em dashboard.js:
   Após c.style.display = 'block', adicionar:
   requestAnimationFrame(() => { void c.offsetHeight; });
   (força repaint sem criar camada GPU)

2. No showView() em theme.js:
   A variável deve se chamar viewEl (não target que não existe)
   Após viewEl.classList.add('active'), adicionar:
   requestAnimationFrame(() => { void viewEl.offsetHeight; });

3. Event delegation nas tabelas:
   Substituir onclick="" inline em cada <tr class="entry-row">
   por um único listener no container pai:
   container.addEventListener('click', e => {
     const row = e.target.closest('.entry-row');
     if (!row) return;
     // lógica do clique
   });
   (tabelas com 50+ linhas ficam muito mais rápidas)

4. No sw.js: adicionar skipWaiting() e limpeza de cache antigo:
   self.addEventListener('install', e => {
     self.skipWaiting();
     e.waitUntil(caches.open(CACHE_NAME).then(c => c.addAll(ASSETS)));
   });
   self.addEventListener('activate', e => {
     e.waitUntil(
       caches.keys().then(keys =>
         Promise.all(keys.filter(k => k !== CACHE_NAME).map(k => caches.delete(k)))
       )
     );
     self.clients.claim();
   });

5. Adicionar content-visibility: auto nas seções de tabela longa
   no tables.css para melhorar performance de scroll

ARQUIVOS:
- js/dashboard.js
- js/theme.js
- sw.js
- css/tables.css

COMMIT: "perf: repaint mobile e event delegation nas tabelas"
